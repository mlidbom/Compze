# Cater to the limitations of the human mind

**Virtually  Everything** in these code standards revolves around one vital insight. 
Software design is about catering to the human mind.

A mind that has a short term memory with something like 5-7 slots. Unlike the thousands of an AI.
Because of various limitations of the human mind, in order to be able to effectively understand and maintain systems of any complexity a human requires:

* a [Ubiquitous Language](007-ubiquitous-language.md)
* [Well encapsulated abstractions](031-everything-in-its-place.md)
* Deep hierarchies of namespaces that make the context and grouping of abstractions clear
* [Long descriptive names](013-name-for-the-human-who-must-look-it-up.md) that make meaning clear without knowing the details

These and the other instructions here are about writing code that allows a human to enter a ubiquitous language into long term memory, 
enabling them to understand, maintain, and change large complex code bases.

## Current state
- Currently our refactoring support requires that one
    - manually creates a class that uses an ITypeMappingRegistrar to register the types in an assembly.
    - given that we will detect any missing mappings at runtime 
        - and generate code for the missing mappings for the user to insert in the existing code file


## Wanted state
- A public interface: Compze.Abstractions.Internal.Refactoring.Naming.IMapCurrentAssemblyTypesForRefactoringSupport 
    - declaring a single method method:
        - void MapTypesForCurrentAssembly(ITypeMappingRegistrar registrar);
- A class named AutoGeneratedCurrentAssemblyTypeMapperForRefactoringSupport implementing IMapCurrentAssemblyTypesForRefactoringSupport must exist in the root namespace(The assembly name) of any assembly containing mapped types
- at runtime when mapping information for a type that is not already mapped is requested from TypeMapper:
    - TypeMapper will extract mapping information for that type's assembly by using reflection to instantiate and invoke the method on AutoGeneratedCurrentAssemblyTypeMapperForRefactoringSupport
        - This will run exactly once for an assembly and the result will be cached
    - if the AutoGeneratedCurrentAssemblyTypeMapperForRefactoringSupport class does not exist
        - throw an except containing the full code required for the file along with instructions for what to name the file and where to place it.
    - if the file exists but is missing a mapping for the current type
         - throw an except containing the full code required for the file along with instructions for what to name the file and where to place it.
         - When generating the file use all existing mappings unchanged so that the mapping for existing types keep using the same IDs and refactoring does not break
    - File placement: The instructions are to place the file visibly in the root folder of the source for the assembly/project
    - How do detect types to map:
        - Mappings should beg generated for every type in the assembly that implements, directly or indirectly any of:
             - Compze.Tessaging.Abstractions.IMessage
             - Compze.Abstractions.IHasPersistentIdentity
